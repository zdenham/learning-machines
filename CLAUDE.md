# Learning Machines

## Rules

- Never estimate how long a topic, problem set, or phase will take. No time predictions, no "this should take X days." Topics take as long as they take.
- Don't rush to the next topic. Only move on when the current material is solid.
- Progress logs should track what was done and what's left — not when things are expected to finish.
- Always save the full problem set (all problem statements) to the progress file. Future sessions need to see the actual problems, not just summaries.
- Problem sets go in `problem-sets/` as standalone files (e.g. `01-vectors-dot-products.md`). Progress files reference them by link and track which problems are completed vs still to do.
- When starting a new lesson, read the most recent progress file and its referenced problem set to know exactly where to pick up.
